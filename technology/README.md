#   资料

技术或应用不以大不以全不以多为重，而是可用，能够解决某类问题

事情开始之前，明确大概的结果，最终产出的效果，然后分割成多个阶段部分，不断迭代，每部分对应不同 的环节

书不是读的、不是看的，是`拆`的，然后`合`起来

`怎样才算掌握了某项技术??`

`怎样快速掌握某项技术??`

先`全部``全部``全部``翻译`.....这是个❌❌错误！！！既然书可以拆，那么技术也是可以拆的

无论是书还是文档都是描述该技术，那么该技术就是结果，找到最短的路线，不要绕圈

1.   思维导图，金字塔
2.   核心元素为节点，大卸八块
3.   针对某主题整体抽象梳理
4.   不纠结细节，不纠结语法，那是API
5.   覆盖、概述、组织
